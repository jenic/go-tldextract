# Security Policy

This security policy applies to the github.com/jenic/go-tldextract project
hosted on GitHub and elsewhere.

## Supported Versions

Security and bug fixes are provided for the last minor version, older versions
are not supported. When a security issue exceeds CVSSv4 8.5 a security fix will
be released immediately. Security fixes that constitute a measurable risk to
the project will be part of the next minor version.

## Reporting a Vulnerability

We encourage responsible disclosure of security vulnerabilities and we
encourage and appreciate your reports. This project does not run an active Bug
Bounty program and is not corporately sponsored. Your contributions will be
credited within a dedicated section of the project with your name and website
of choice, assuming they are appropriate to publicly display. We reserve the
right to refuse crediting obscene or inappropriate names or websites.

### Preferred Reporting Method

Please report vulnerabilities using the GitHub security tab's Report a
Vulnerability button or by contacting the developers directly. Do not report
vulnerabilities as a normal GitHub issue.

### Information to Include

To help us understand the nature and scope of the issue, please include:

* Type of issue (e.g. Buffer overflow, Path Traversal, etc)
    * Where possible, utilize CAPEC terminology to help standardize language:
      capec.mitre.org
* Location (affected source file(s), branch, functions, etc)
* Step-by-step instructions to reproduce the issue.
* Impact and potential attack chains or exploit scenarios.

### Our Process

Once a report is received:

1. **Acknowledgment:** We will acknowledge your report within 72 hours, if you
   have not gotten a response from us within 72 hours please follow-up using a
   different outlined communication method.
2. **Verification:** We will attempt confirmation of the vulnerability via
   replication or technical investigation.
3. **Fix:** We will work on a patch for supported version(s) and, should you
   desire it, work with you to verify the fix.
4. **Disclosure:** We follow a 30 day disclosure timeline before public
   announcement. Please do not post publicly about the issue until people have
   had time to patch their systems.

We appreciate your efforts to keep our users safe through responsible
disclosure.
